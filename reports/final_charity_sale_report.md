# Final Charity Sale Report

## Overview

This report summarizes an anonymized charity sale data analysis project. The
records include direct donations, donated item inventory, sale records, and
booth planning notes. In this sample analysis, direct donations and sale
revenue together add up to ¥27,947, which is over ¥26,000.

This report is not an official financial report or audit. It is a student data
analysis and event review project based on anonymized sample records.

## Event Background

The community charity sale included direct donations and item sales. I
supported the data and operations side by counting donated items, categorizing
inventory, estimating item values, maintaining online spreadsheets, helping
with item allocation, supporting booth layout planning, tracking preparation
progress, and reviewing post-event data.

## My Data and Operations Role

- Counted and categorized donated items
- Estimated item values before the sale
- Maintained online spreadsheets
- Supported item allocation across teams
- Helped organize booth layout planning
- Tracked preparation progress
- Reviewed post-event records
- Built this anonymized Python analysis project after the event

## Data Sources

- Donation records
- Item inventory records
- Sale records
- Booth layout records
- Processed merged event data

## Main Results

- Total direct donations: ¥22,678
- Total sale revenue: ¥5,269
- Total funds raised in the sample analysis: ¥27,947
- Direct donations made up the larger part of the total funds
- Shared Table and Booth A were important areas for smaller items

## Donation Analysis

The donation analysis groups direct donations by donor type and team. Larger
direct donations from families and local businesses played an important role in
the total amount, while smaller individual donations also helped broaden
participation.

## Inventory Analysis

The inventory analysis groups item quantities and estimated values by category.
This helped show which categories required more preparation time and table
space, especially snacks, handmade crafts, stationery, and accessories.

## Sales Analysis

The sales analysis summarizes revenue by item category, team, and booth area.
This helped compare which categories and booth areas performed better during
the sale.

## Booth Layout Analysis

The booth layout analysis compares assigned team, main category, planned item
counts, actual item counts, and booth revenue. This was useful for reviewing
table allocation and thinking about how future events could arrange items more
clearly.

## Estimate vs Actual Price

The estimate vs actual comparison connects inventory records with sale records
through item IDs. It shows how final buyer behavior sometimes differed from
pre-event estimated values.

## Simple Machine Learning Experiments

I added two small machine learning experiments after the main analysis. The
price prediction model compares estimated item values with final sale prices.
The sale success model checks which item features were connected with sold or
unsold outcomes.

These models are exploratory learning experiments. They are not official
decision-making tools because the dataset is small and anonymized.

## Operational Insights

- Clear item categories made booth planning easier.
- Item IDs helped connect inventory and sale records.
- Team-level summaries made post-event review easier.
- Small, lower-priced items needed enough table space because they sold in
  higher quantities.
- Comparing estimated value and sale price helped review pricing choices.

## Limitations

- The data is anonymized and sample-based.
- This is not official financial accounting.
- Booth traffic was not fully recorded.
- Some estimated item values may differ from buyer behavior.
- The machine learning models are exploratory and should not be over-read.

## Future Improvements
